OOP Concepts

Object-Oriented Programming (OOP) organizes code into classes and objects, making it modular, reusable, and easy to maintain. The four pillars of OOP are:

1. Encapsulation

Wrapping data (fields) and behavior (methods) together in a single unit (class).

class Student { private String name; public void setName(String n) { name = n; } public String getName() { return name; } }

2. Inheritance

One class can inherit fields and methods from another.

class Animal { void sound() { System.out.println("Animal makes a sound"); } } class Dog extends Animal { void sound() { System.out.println("Dog barks"); } }

3. Polymorphism

Same method name but different behavior depending on context.

Animal a = new Dog(); a.sound(); // Output: Dog barks

4. Abstraction

Hiding implementation details and showing only essential features.

abstract class Shape { abstract void draw(); } class Circle extends Shape { void draw() { System.out.println("Drawing Circle"); } }

Summary

OOP principles—Encapsulation, Inheritance, Polymorphism, and Abstraction—form the foundation of modern programming in Java, Python, C++, and many other languages.
